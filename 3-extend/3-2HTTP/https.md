# 一、为什么需要三次握手
1、创建可靠连接的最小握手次数。  
2、数据的发送都是基于序列号（SEQ）的，不同的计算机初始序列号（ISN）不同；  
计算机A、B间需要创建连接，那么A必须知道B的ISN，B也必须知道A的ISN；  
第一次握手 A -> (SYN) -> B，B会知道A的ISN号；  
第二次握手 B -> (ACK,SYN)-> A，B向A确认收到的A的ISN号，并发送B自身的ISN号给A；  
第三次握手 A -> (ACK) -> B，A向B确认收到了B的ISN号。  
如果只用两次握手，那么B无法判断 A是否已经收到自己的ISN号，无法创建连接。

# 二、为什么需要四次挥手
因为TCP是双全工通道（两方都能发送、接收数据），每个方向都需要单独关闭。
发送数据和接收数据是两码事，当一方认为自己不再需要发送数据时候会发送一个FIN来终止这个方向的
连接，当一方收到一个FIN请求时，意味这不会再收到数据，但是可以继续发送。
（1）A不再发送数据，向B发送一个FIN请求；
（2）B收到FIN后返回一个ACK，但B还可以继续发送数据；
（3）当B也不再发送数据时，向A发送一个FIN；
（4）A收到FIN后返回一个ACK。
